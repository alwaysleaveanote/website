Lists and Tables

Lists

There are ordered lists (< ol >) which are numbered, and unordered lists (< ul >) which are bulletted.
You define an element of a list using the in line < li > tag. For example:

  • Unordered list element!
  • Another unordered list element!
  • Woah, gonna make an ordered list inside an unordered list!
    1. One!
    2. Two!
    3. Three!

Along with ordered lists and unordered lists, there are also definition lists
essentially uses the same syntax, except < dl > initializes the list, < dt > are the terms,
and < dd > are the definitions of those terms. For example:

This is a term.
This is a definition. It's indented!
You can put more than one definition!
Another Term!
Only one definition this time.

Tables

Use the < table > tag to initiate a new table.
Within a table, use < tr > to create a new row.
Within in a table row use < td > to create a new cell.
Elements are not broken up by anything (commas, brackets, lines, etc.)
For example:

(1, 1) (1, 2) (1, 3)
(2, 1) (2, 2) (2, 3)
(3, 1) (3, 2) (3, 3)
(4, 1) (4, 2) (4, 3)

can define headers for headers using the < th > tag. It is used the same and looks the same as < td >
but means that the data in that cell is a header the column, not an actual piece of data.
the colspan and rowspan attributes essentially merge two cells one column or one row apart together. For example:

header 1 header 2 header 3
(2, 1) (2, 2 AND 3)
(3 AND 4, 1) (3, 2) (3, 3)
(4, 2) (4, 3)

To change the formatting of a column, use the < colgroup > (not inline) and < col > (inline, unpaired) tags.
withing a < colgroup > tag, the first < col > element is the first column, the second < col > element is the
second column, etc. < col > tags can have a class attribute (class = "alternative" for example) and a span attribute
which combines 2 or more columns (span = "2"). It is similar to < colspan > except works for every row.

(1, 1) (1, 2 AND 3)
(2, 1) (2, 2 AND 3)
(3, 1) (3, 2 AND 3)
(4, 1) (4, 2 AND 3)

You can also add a caption to your table by using the < caption > (inline, paired) tag right after
you declare the opening < table > tag.

To make the table easier to read in HTML for, you can use the < thead >, < tbody >, and < tfoot > tags. < thead > always comes first, but < tfoot > can come before < tbody > (though it will be rendered after < tbody >)
